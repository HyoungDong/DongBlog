                                    CSS Flex 기술이 나오기 전에는 float 속성을 사용해 수평 정렬을 했습니다.
                                    수평(수직) 정렬을 위한 Flex 기술이 나오고 나서도 브라우저 하위호환성을 이유로 꽤 오랫동안 float을 사용해 왔습니다만,
                                    근본적으로 float은 수평 정렬을 위한 속성이 아닙니다.
                                    하위호환성 이슈가 많이 사라진 현 상황엔 강의에선 Flex을 이용한 수평 정렬로 교육을 하는 것이 맞습니다만,
                                    아직도 수많은 사이트가 float을 이용한 수평 정렬로 구현되어 있어서 그 교육의 기준이 모호한 것도 사실입니다.
                                    그래서 결론은,
                                    수평 정렬을 위한 CSS 기술은 Flex가 정답입니다.
                                    대신 예전에 만들어진 사이트의 유지/보수를 위해 float을 사용한 수평 정렬도 알아두는 것이 좋습니다.
                                    참고로 Flex는 호환성 작업을 거치면 IE11 버전에서도 사용할 수 있습니다!

                                                                                                    정확하게 보셨고 우선 결론만 말씀드리면 위 예제에서 <div class="container clearfix">의 clearfix는 필요하지 않습니다.
                                                                                                    그런데 clearfix가 있는 이유를 말씀드리면,
                                                                                                    입문자 편에서 완성한 위 코드들의 결과는 다른 영역의 콘텐츠 작업을 모두 생략했고(전체 예제의 일부분만 만들었죠) 그러다보니 원래는 CSS 코드에 선택자 .container-left 부분에 float: left가 적용되어야 하는데 당장의 예제에는 필요치 않아 주석 처리(/* float: left; */ 이렇게요)했습니다.
                                                                                                    만약 .container-left 부분에 float: left가 적용되었다면 clearfix가 필요했겠죠.
                                                                                                    그래서 남아있는 흔적이라고 생각하시면 됩니다.
                                                                                                    참고로 이렇게 clearfix가 필요치 않은 곳에 적용되었다고 하더라도 레이아웃에 미치는 영향은 없기 때문에 문제는 없으며, 'clearfix가 필요할 수도 있는 곳에 그저 지워지지 않고 남아있다' 정도로 이해하시면 좋을 것 같습니다.
                                                                                                    그래서 남겨두셔도 되고 지우셔도 됩니다 :D

                                                                                                                                            CSS 강의 자료는 내용이 워낙 많아 따로 블로그에 정리하지 않았습니다.
                                                                                                                                            온라인 강의에서 사용한 자료는 https://github.com/HeropCod... 에서 확인하실 수 있고, 온라인 강의와 전혀 상관없는 예전에 개인적으로 사용하던 자료이기 때문에 커리큘럼은 일치하지 않습니다.
                                                                                                                                            위 링크에 없거나 정리가 더 필요한 내용 일부만 블로그로 옮겼으니 참고하세요.
